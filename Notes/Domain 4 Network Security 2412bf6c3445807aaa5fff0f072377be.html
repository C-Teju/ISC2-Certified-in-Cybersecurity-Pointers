Domain 4: Network Security

Networking

What is Networking?

Networking refers to the practice of connecting two or more computers or devices to share data, information, or resources such as files, printers, or internet access. A network allows these connected devices to communicate and collaborate efficiently.

To establish secure data communications, it's important to understand the technologies involved, including hardware, software, protocols, and encryption.

There are two basic types of networks:

Local Area Network (LAN): A LAN typically spans a single floor or building and covers a limited geographical area.

Wide Area Network (WAN): A WAN refers to long-distance connections between geographically remote networks.

Network Devices

  • Hubs: Hubs connect multiple devices in a network, mainly in home setups. They are wired and less advanced than switches or routers.
  • Switch: A switch is a wired, intelligent hub that sends data only to the intended device, improving efficiency and throughput. It's smarter than a hub but not as smart as a router. Switches are wired devices that know the addresses of the devices connected to them and route traffic to that port/device rather than retransmitting to all devices. Switches can also create separate broadcast domains using VLANs.
  • Router: Routers manage traffic flow between networks, connecting multiple switches. They can be wired or wireless and choose the most efficient path/route for data, making them smarter than hubs and switches.
  • Firewall: A firewall filters and controls network traffic to protect the network. It‚Äôs usually placed between a private network and the internet or between departments within an organization, using defined rules or access control lists.
  • Server: A server is a computer that provides information to other computers on a network. Examples include web, email, print, database, and file servers. They are accessed by client computers and are secured differently than workstations to protect their data.
  • Endpoint: Endpoints are the ends of a network communication link, typically a client device requesting resources from a server. They can include servers, desktops, laptops, tablets, phones, or any end-user device.

Other Networking Terms

  • Ethernet: Ethernet (IEEE 802.3) is a standard that defines wired connections between networked devices. This standard defines how data is formatted over the wire to ensure different devices can communicate over the same cables.

Device Address

  • Media Access Control (MAC) Address: Every network device has a MAC address, e.g., 00-13-02-1F-58-F5. The first 3 bytes identify the manufacturer. No two devices in the same local network can have the same MAC address, or an address conflict will occur.
  • Internet Protocol (IP) Address: MAC addresses are built into hardware, while IP addresses are logical and help identify the device on a network. IP addresses allow continued communication even if the physical hardware changes. Examples include 192.168.1.1 and 2001:db8::ffff:0:1

Microsegmentation

Microsegmentation helps protect data centers from modern cyberattacks that can bypass traditional security. It works by dividing the network into smaller, secure segments and controlling traffic within and between them. This approach improves threat detection and makes it harder for attackers to move across systems. Avoiding infrastructure-focused designs also boosts efficiency and security.

Microsegmentation Characteristics:

  • Allows detailed control over traffic, down to individual users, devices, times, credentials, and services.
  • Uses logical rules (not physical ones), requiring no extra hardware or manual device access.
  • Supports defense-in-depth by preventing one compromised point from affecting the entire network.
  • Essential in cloud/shared environments to protect data from unauthorized access, including by cloud provider staff.
  • Enforces least privilege by limiting communication between departments (e.g., isolating HR data).
  • Enabled by virtualization and software-defined networking (SDN); uses tools like VPNs or security groups.
  • Can even be used at home to separate smart devices from personal computers.

Tools to Identify and Prevent Threats

Malware (Malicious Software): A general term for any software designed to harm, exploit, or steal data from devices or networks. This includes viruses, worms, trojans, ransomware, spyware, and more.

Here are definitions for common types of malware:

  • Virus: Malicious code that attaches to files or programs and spreads when the infected file is opened. It can damage or delete data.
  • Worm: A standalone program that spreads itself across networks without needing to attach to other files. It often consumes bandwidth or overloads systems.
  • Trojan (Trojan Horse): Malware disguised as a legitimate program. Once installed, it can give attackers access to the system or perform harmful actions.
  • Ransomware: Malware that locks or encrypts your data and demands payment (ransom) to restore access.
  • Spyware: Software that secretly monitors user activity, often capturing sensitive data like passwords or credit card numbers.

These all fall under the broader category of malware, designed to disrupt, damage, or gain unauthorized access to systems.

DoS Attack (Denial of Service):

A DoS attack floods a system (like a website or server) with so much traffic that it becomes slow or unavailable to real users.

üîπ Goal: Overwhelm the system so it crashes or can't respond.

üîπ Example: Too many fake login requests make the website crash for everyone.

Spoofing:

Spoofing is when an attacker pretends to be someone else‚Äîlike faking an IP address, email, or website‚Äîto trick users or systems.

üîπ Goal: Gain access or fool the victim.

üîπ Example: A fake email looks like it‚Äôs from your bank and asks for your password.

Side-Channel Attack:

This attack doesn‚Äôt directly break into a system, but instead gathers information from things like power usage, timing, or electromagnetic leaks to guess secrets (like passwords or encryption keys).

üîπ Goal: Steal data indirectly.

üîπ Example: Measuring how long a system takes to respond in order to figure out a password.

Intrusion Detection System (IDS)

An IDS monitors logs and real-time events to detect abnormal activity or intrusion attempts. It automates threat detection and supports other security tools like firewalls as part of a defense-in-depth strategy but does not replace them.

IDSs detect both external and internal attacks, like internet-based threats or spreading worms. They respond by raising alerts for timely action. IDS helps isolate secure zones from less trusted ones and often complements firewalls.

  • There are two main types: Host-based IDS (HIDS): Monitors a single device and Network-based IDS (NIDS): Monitors network traffic patterns.

    Host-based Intrusion Detection System (HIDS):

    A HIDS monitors activity on a single computer, including logs and processes, and can detect detailed host-level attacks like remote control or file compromise. It's better at spotting local anomalies than NIDS but is costlier to manage since it requires setup on each system. It cannot detect attacks on other devices in the network.

    Network Intrusion Detection System (NIDS):

    A NIDS monitors network traffic for attacks or anomalies. It can‚Äôt see inside encrypted data but analyzes packet details. Using sensors at key points like routers or firewalls, it collects data for central analysis. NIDS has minimal impact on network performance but may not show if an attack was successful or what it affected.

Security Information and Event Management (SIEM):

SIEM refers to tools that collect and analyze security data from multiple sources across an organization‚Äôs IT environment. These tools generally known as security information and event management (or SI-E-M, pronounced ‚ÄúSIM‚Äù) help identify potential threats, streamline security efforts, and improve overall visibility into security posture. SIEM solutions gather log data from systems, applications, and devices to detect anomalies and support faster incident response. They are often used as part of a broader defense-in-depth strategy to strengthen an organization‚Äôs information security program.

Preventing Threats

While no single method can stop all threats, some basic practices help reduce risk:

  • Keep systems updated: Apply patches regularly to fix bugs and security flaws. Use patch management to stay current.
  • Disable unneeded services/protocols: Only run what's necessary to reduce attack surfaces.
  • Use IDS/IPS: Intrusion Detection and Prevention Systems monitor and detect threats, and can block attacks.
  • Use firewalls: Network firewalls protect entire networks, while host firewalls guard individual systems.
  • Use updated anti-malware software: Keeps systems protected from viruses and other malicious software.

Antivirus:

Antivirus software is a key security practice and is required for PCI DSS compliance. It identifies malware using known signatures or by detecting abnormal behavior through scanners, pattern recognition, and machine learning. Modern anti-malware solutions go beyond viruses, detecting threats like rootkits, ransomware, and spyware. Many enterprise solutions also include firewalls and IDS/IPS as part of endpoint protection.

Scans:

Regular vulnerability and port scans help assess the effectiveness of an organization‚Äôs security controls. They can uncover missing patches, weak settings, new or exposed vulnerabilities, and non-compliance with security policies‚Äîall of which attackers may exploit.

Network Segmentation: Demilitarized Zone (DMZ)

A DMZ (Demilitarized Zone) is a network segment that acts as a buffer between an external network (like the internet) and an internal network. Systems in the DMZ, such as web servers, are isolated using firewalls and secured switches to limit access and protect internal resources. This setup helps implement defense in depth, allowing only legitimate traffic to reach internal systems while reducing the risk of external attacks.

DMZ (Demilitarized Zone) Deeper Dive

üîê DMZ (Demilitarized Zone) ‚Äì Deep Dive Explained

A DMZ (Demilitarized Zone) is a buffer zone between an external network (like the internet) and the internal secure network. It is commonly used to host public-facing services, such as web servers, email servers, or DNS servers, while limiting access to internal systems.

‚úÖ Example Architecture:

  • A web front-end server is placed in the DMZ, accessible from the internet.
  • This web server retrieves data from a database server that resides in the internal network, which is protected by a second firewall.
  • This setup ensures that even if the web server is compromised, the attacker cannot directly access sensitive internal systems or data.

‚úÖ Real-World Example:

In a hospital or healthcare setting:

  • A segregated network may handle patient information and billing, kept behind an internal firewall.
  • A web-based EMR (Electronic Medical Records) application may be deployed in the DMZ to allow controlled external access.
  • Internally, a secure database server stores sensitive health records and is only accessible through tightly controlled rules set by authorized personnel via the firewall.
  • Secure switches or access controls ensure only designated personnel can manage firewall rules or network segmentation.

üîç WAF (Web Application Firewall) vs DMZ

  • A WAF is not a substitute for a DMZ, but rather an additional security layer.
  • A Web Application Firewall (WAF) filters and monitors HTTP/HTTPS traffic to and from a web application.
  • It protects against common web threats such as SQL injection, cross-site scripting (XSS), and more‚Äîeven if the traffic is encrypted.
  • The WAF sits in front of the web server, often behind a traditional firewall, and analyzes requests before passing them to the application server.

Security of the Network

  • üîê Security of the Network ‚Äì Understanding TCP/IP Vulnerabilities

    The TCP/IP protocol suite, which is the foundation of communication over the internet and most networks, was not originally designed with strong security in mind. As a result, it has several vulnerabilities that attackers can exploit, especially when the TCP/IP stack is improperly implemented in an operating system.

    üö® Common TCP/IP Vulnerabilities:

    1. DoS/DDoS Attacks (Denial of Service / Distributed Denial of Service)
      • Attackers flood a system or network with excessive traffic to exhaust resources and make services unavailable.
    1. Fragmentation Attacks
      • Attackers manipulate how packets are fragmented and reassembled to evade detection or crash systems.
    1. Oversized Packet Attacks
      • Sending larger-than-normal packets can crash or destabilize network devices with poor input validation.
    1. Spoofing Attacks
      • In spoofing, an attacker falsifies the source IP address of packets to trick a system into believing the data is from a trusted source.
    1. Man-in-the-Middle (MitM) Attacks
      • The attacker intercepts and possibly alters communication between two parties without their knowledge.

    üïµÔ∏è‚Äç‚ôÇÔ∏è Passive Attacks ‚Äì Sniffing or Monitoring

    Even if active attacks are prevented, passive attacks can still occur:

    • Sniffing (Network Monitoring):

      This involves passively listening to network traffic to capture sensitive data like usernames, passwords, or session tokens.

      Tools like Wireshark or tcpdump can be used for legitimate analysis‚Äîbut also misused for spying.

    üß± Mitigation Strategies:

    • Use network segmentation and firewalls.
    • Encrypt data in transit using TLS/SSL and VPNs.
    • Deploy intrusion detection/prevention systems (IDS/IPS).
    • Regularly update operating systems and firmware to patch known vulnerabilities in the TCP/IP stack.
    • Use strong authentication and access controls.

SYN, SYN-ACK, ACK Handshake / 3-Way Handshake

Before a client and server can communicate over TCP (Transmission Control Protocol), they must establish a reliable connection. This process is called the three-way handshake.

  • Let‚Äôs break it down in a simple scenario‚Äîwhen a client (like your browser) wants to connect to a web server (like a website on port 80 for HTTP or port 443 for HTTPS):

    üîÅ The Steps of the 3-Way Handshake:

    1. SYN (Synchronize)

      The client sends a TCP packet with the SYN flag set to the server's IP address and port 80 or 443.

      üëâ This means: "I want to start a connection and here‚Äôs my initial sequence number."

    1. SYN-ACK (Synchronize-Acknowledge)

      The server responds with a packet that has both the SYN and ACK flags set.

      üëâ This means: "I got your request. Here‚Äôs my sequence number, and I acknowledge yours."

    1. ACK (Acknowledge)

      The client sends back an ACK packet to acknowledge the server's sequence number.

      üëâ This means: "Acknowledged. We‚Äôre now connected!"

    ‚úÖ What Happens Next?

    After this handshake is complete, a TCP connection is established. The client and server can now begin secure or insecure data exchange, depending on the protocol (e.g., HTTPS or HTTP).

    üìå Why It Matters:

    • Ensures both parties are ready to communicate.
    • Sets up sequence numbers for tracking data.
    • Prevents data loss or corruption during transmission.

Cloud Redundancy

Organizations are moving to cloud-based systems instead of maintaining physical server rooms because the cloud offers better flexibility, security, and reliability.

Cloud providers (like AWS, Azure) have multiple availability zones. If one zone fails, the system automatically switches to another, ensuring no downtime ‚Äî this is called cloud redundancy.

You don‚Äôt need to build or maintain your own data center; the provider handles it for you.

Cloud services are usually pay-as-you-go, and resources like storage and processing are shared (pooled) across users, making it cost-efficient and scalable.

Service Models

Cloud services come in three main models, each with different levels of responsibility:

  • SaaS (Software as a Service): The provider manages everything ‚Äî you just use the software (e.g., Gmail, Microsoft 365).
  • PaaS (Platform as a Service): You manage the apps and data; the provider handles servers and infrastructure (e.g., Google App Engine).
  • IaaS (Infrastructure as a Service): You control the OS, apps, and data; the provider only supplies the raw infrastructure (e.g., AWS EC2).

Organizations must ensure security (especially for data) based on which model they use, as responsibility varies between provider and user.
